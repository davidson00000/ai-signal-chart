残りの2割が “勝つための本質コア” の部分。

🟥 1. 銘柄選定AI（スクリーナー）を作る — 最優先

勝つために一番重要なのは「何を監視すべきか」。
これをAIにやらせる。

🎯 目的

毎朝「今日見るべき銘柄リスト」を生成する。

勝率の高い銘柄だけを監視する。

🛠 タスク

 市場データから特徴量を抽出（RSI, MA, ATR, MACDなど）

 ボラティリティ・トレンド性・出来高などを統合した“優位性スコア”作成

 大量銘柄（S&P500など）に対して毎日スクリーニング

 監視用銘柄リストを /recommended-symbols API で返す

 Mobile Live View に反映（監視銘柄の表示）

✨ 期待する結果

“勝てる銘柄だけ見る” 状態が作れる
＝勝率が最も大きく上がる。

🟧 2. リスク管理エンジン（1R / ロット計算 / 損切りライン）

バックテストでは強くても、実運用ではリスク管理が全て。

🎯 目的

1トレードの最大損失（1R）を自動計算

ATRなどから損切りライン推奨

最適ロット数を自動提示

🛠 タスク

 1Rの定義設定（ユーザーの資金に基づく）

 ATRベースで損切りラインをAIが決定

 損切りラインから買付可能株数を計算

 シグナル詳細に「推奨ロット」「損切りライン」を追加

 Mobile Live View に反映

✨ 期待する結果

資金曲線が“安定”する。
勝つシステムは負け方が上手い。

🟨 3. 市場レジーム判定（トレンド or レンジ）

市場環境と戦略が噛み合わないと負ける。

🎯 目的

市場が「強トレンド」「弱トレンド」「レンジ」を自動判定

各戦略の優先順位を動的に変える

🛠 タスク

 ADX, MACD, ATR を使った環境認識AI

 /market-regime API の実装

 Strategy Lab にレジーム情報を統合

 Live Signals の戦略選択に反映

✨ 期待する結果

相場に適した戦略だけが作動する。
ミスマッチによる負けトレードが減る。

🟩 4. メタ戦略（戦略投票システム）

複数戦略の意見を統合して最終判断を決める。

🎯 目的

MA/RSI/MACD など複数戦略の判断を統合

Confidence（信頼度）をより正確にする

🛠 タスク

 戦略ごとのBUY/SELL判定をまとめる投票ロジック

 戦略の重み付け（勝率・DD・期待値）

 Explainability の “meta_explain” を実装

 Live Signal が単戦略ではなく“統合シグナル”を出すように変更

✨ 期待する結果

シグナルの精度が跳ね上がる。
ダマシが減り、プロの判断に近づく。

🟦 5. 通知システム（Push / LINE / Email）

準自動に最も必要な部分。
スマホでアラートを受け取り“そのまま注文”につなげる。

🎯 目的

シグナルが出た瞬間、スマホに通知

通知の中に「理由」「ロット」「損切り」を含める

🛠 タスク

 Push通知API（Pushbullet / LINE Notify）

 アラート用FastAPIエンドポイント作成

 Mobile Live View と連動

 通知フォーマットの標準化

✨ 期待する結果

外出先で即判断 → 準自動投資が実用化。

🟪 6. ポートフォリオ管理（複数ポジションの最適化）

単銘柄で勝てても、複数を持つと破綻しやすい。
ここを入れると EXITON が「運用ツール」になる。

🎯 目的

同時ポジションの相関性を管理

全体のリスク上限を設定

🛠 タスク

 ポジションの相関分析（β, Sector, Correlation）

 最大DDの制御

 1日の最大損失ルール（Daily Stop）

 /portfolio-risk API の実装

✨ 期待する結果

トレードではなく“投資”として勝てるようになる。

🟫 7. フィードバック学習（戦略の優先度の動的調整）

プロの準自動システムが必ず持つ要素。

🎯 目的

過去数日の結果から“今強い戦略”を見極める

戦略ごとの勝率とDDから重みを調整

🛠 タスク

 トレード結果の自動記録

 戦略別パフォーマンスの集計

 重み付けの動的更新（週次 or 日次）

 meta_strategiesに反映

✨ 期待する結果

EXITONが“学習するシステム”になる。
